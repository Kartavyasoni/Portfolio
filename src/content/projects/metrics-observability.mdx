## Overview

This is the project that turns "the dashboard looks wrong" into "the suite
already flagged it and opened an incident." It spans the full stack — from dbt
tests to a React lineage graph.

## Lineage as the backbone

Column-level lineage answers the only question that matters during an incident:
*what upstream change caused this, and what downstream depends on it?*

## Design choices

- Treat data tests as code, versioned alongside the models they validate.
- Anomaly detection on freshness and row counts catches silent failures.
- Alerts are actionable: they link straight to the failing model and its lineage.
