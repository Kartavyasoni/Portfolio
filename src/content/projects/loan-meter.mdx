## Overview

Loan Meter is a credit-risk model designed for the real world: imbalanced data,
a hard precision requirement, and a regulatory need to **explain every
decision** — not just produce a score.

## Handling imbalance honestly

With a ~10:1 class imbalance, naive accuracy is meaningless. I used
**cost-sensitive learning** to make the model pay attention to the rare default
class, lifting minority-class recall by 14% without collapsing precision.

## Tuning for the right metric

The XGBoost classifier was tuned with **Bayesian hyperparameter search** toward
ROC-AUC and precision on approvals — reaching 0.76 ROC-AUC while keeping
false approvals low.

## Explainability as a requirement

Every prediction carries **SHAP reason codes** — *"income-to-debt ratio,
recent delinquencies"* — so a denial can be explained to an applicant and a
regulator. Transparency was a design constraint, not an afterthought.
