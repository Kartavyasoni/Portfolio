## Overview

A feature store is really a **contract**: the same feature, computed the same
way, everywhere. That contract is what keeps offline metrics predictive of
online behavior.

## Point-in-time correctness

Training sets are assembled with point-in-time joins so models never see future
information — the subtle bug that inflates offline metrics and disappoints in
production.

## Operations

- Stateful Flink jobs handle late and out-of-order events.
- Redis serves the online store with strict latency budgets.
- Feature definitions are versioned and reviewed like any other code.
